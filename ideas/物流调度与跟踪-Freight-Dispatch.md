# 物流调度与跟踪 (Freight Dispatch AI — 小型货运经纪商)

## 核心价值主张

**"你的货运经纪公司需要 AI 替代每天 80% 的重复电话。"**

小型独立货运经纪商（1-10 人）每天的工作几乎全是信息流：
- 接货主来电（需要运什么/什么时候/去哪里）
- 联系司机（有没有空车？价格？）
- 调度确认（取货/送货时间/路线）
- 跟踪更新（货物到哪了？延误了吗？）
- 结算催款（Invoice 对账/付款跟进）

**AI 能全部接管，小经纪商只需处理例外情况。**

## 目标客户

- 1-10 人的独立货运经纪公司
- 每月处理货运量：100-1000 票
- 年营收：$500k-$5M
- 痛点：人手少，24/7 接不完电话，司机突然取消很麻烦
- 付费能力：一个调度员 $3k-4k/月，AI 替代有 ROI

## AI 员工团队设计

### Agent 1：货主接单 Agent
- 接听货主询价（电话/邮件/Web表单）
- 收集关键信息：货物/重量/起点/终点/时间要求
- 自动生成初步报价（基于历史数据）

### Agent 2：司机协调 Agent  
- 在 DAT/Truckstop 发布货源
- 筛选符合条件的司机（评级/资质/位置）
- 自动发 SMS 确认细节

### Agent 3：调度跟踪 Agent
- 出发/中途/到达自动提醒
- 延误预警（超过预计时间 → 立即通知）
- 货主自助查询（"我的货到哪了？"）

### Agent 4：结算 Agent（Phase 2）
- 自动生成 Invoice
- 跟踪付款状态，逾期自动催款

## 竞品差异

| 竞品 | 做什么 | 差距 |
|------|--------|------|
| DAT/Truckstop | 信息板（找货找车）| 工具，不是 AI Agent |
| Loadsmart | 大型托运商 AI 定价 | 服务大企业，不是小经纪商 |
| Convoy | 数字货运（已倒闭）| 教训：不能完全去掉人 |

**Renaix 的定位：不替代经纪商，而是给经纪商配 AI 员工**

## 行业背景

- Convoy 2023 年倒闭（$1B+ 融资）— 教训：自动化过度，市场不接受
- 这反而是 Renaix 的机会：不是颠覆，而是增强小经纪商的能力
- 小型独立经纪商 1.5 万+ 家，几乎没有 AI 渗透

## 商业模式

- Setup：$8,000-$15,000（系统集成 DAT/TMS/电话）
- 月费：$2,000-$5,000/月（基于处理票数）
- 首收时间：6-10 周

## GTM 深化执行（2026-02-21 补充）

### 渠道优先级重排

基于 GC 产品的 GTM 经验迁移，货运经纪商的信任来源如下（从高到低）：

1. **DAT/Truckstop 社区**（货运经纪商每天使用的平台，论坛活跃）
2. **TIA 行业协会成员推荐**（小型经纪商信任 TIA 背书）
3. **LinkedIn 同行推荐**（货运经纪人有强 LinkedIn 文化，比 GC 更活跃）
4. **FreightWaves/The Loadstar 媒体**（行业媒体有思想领袖背书价值）
5. **冷外联（电话/邮件）**（效果最差，但可以规模化）

### 渠道 1：DAT / Truckstop 社区渗透

**执行：**
- DAT 社区论坛（载货方和经纪商都有）发起「研究访谈」帖
- 话题切入：「我在研究小型货运经纪商如何处理司机取消导致的紧急调度问题」
- 每周参与 3-5 个帖子的讨论（建立存在感），2 周后开始发现访谈邀请

**筛选标准：**
- 帖子中明确提到「人手不够」「晚上电话太多」「司机取消很烦」的经纪商
- 月处理量 100-500 票（太小不够稳定，太大有内部技术团队）

### 渠道 2：TIA 年会（Transportation Intermediaries Association）

**时间：** TIA 年会每年 4 月举行（Las Vegas）
**目标行动：**
- 以「技术供应商 Associate Member」身份参加（年费约 $1,500）
- 在 Exhibitor Hall 展位（小展位约 $3,000-5,000）演示 AI 调度工具 Demo
- 目标：现场收集 30+ 名片，返回后 1 周内发现访谈邀请

**话术（TIA 展位）：**
> "你们每天花多少时间在电话上处理司机确认和货主状态查询？
> 我们的 AI 可以把这个减少 70%——我不是说完全自动化，
> 而是让 AI 处理重复的部分，你的团队只处理真正需要判断的例外情况。
> 能给我 10 分钟演示一下吗？"

### 渠道 3：「Convoy 后遗症」内容营销

**背景：** Convoy 2023 年倒闭是货运行业的重大心理事件，经纪商既庆幸「没被颠覆」，又担心「AI 会不会再来」。

**内容策略：**
在 FreightWaves 撰投稿（或 LinkedIn 文章）：

**文章标题：**「Convoy 倒闭一年后：小型货运经纪商如何用 AI 增强自己，而不是被 AI 替代」

**核心论点：**
- Convoy 错了：试图用 AI 完全替代经纪商（删除人的判断）
- 正确做法：用 AI 替代经纪商的重复工作，保留人的判断
- 案例：一个 3 人经纪公司用 AI Agent 处理 80% 的状态查询和司机确认，团队专注于谈判和例外处理 → 处理量增加 40%，团队规模不变

**分发渠道：**
- LinkedIn（货运经纪人有强 LinkedIn 文化）
- FreightWaves 投稿
- DAT 社区论坛粘贴摘要 + 链接

### 发现访谈筛选问题（货运版）

（从 GC 版 20 问脚本迁移适配）

1. 你们公司现在几个人处理调度？每人每天大概接多少个状态查询电话？
2. 司机突然取消时，你们是怎么处理的？通常要花多久解决？
3. 货主会在非工作时间联系你们吗？你们怎么处理？
4. 有没有因为沟通不及时，丢掉过货或者被货主投诉的经历？
5. 你现在用什么 TMS（Transport Management System）？满意吗？
6. 如果有工具能自动回答货主「货到哪了」的问题，你觉得每月值多少钱？

## 更新评分：8.0/10（原 7.2）

**提升理由：**
- GTM 渠道更清晰（DAT 社区 + TIA 年会 + 内容营销）
- Convoy 倒闭给了精准的市场教育机会（「我们不是 Convoy」）
- 与 GC 产品技术复用率高，实际开发成本低于原预估
- 单位经济学极佳（LTV/CAC 22-54x）

**维持 8.0 不升至更高的原因：**
- 货运行业 TMS 集成复杂度高（TMW, McLeod, Aljex 等系统 API 难）
- 需要 DAT API 合作协议（非公开，需要洽谈）
- 行业术语和法规学习曲线陡峭（FactorReady、MC 号、BOL 等）

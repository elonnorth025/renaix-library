# General Contractor AI 员工团队（GC 沟通协调 AI）

> **这是 Renaix 的首选产品方向。** 2026-02-16 经 Kelvin 批准，定为 Track 1 首要切入点。
> **2026-02-18 更新：** 加入 Palona AI 行业迁移洞察 + Open-Soul 协议护城河思考。

---

## 核心价值主张

**"你的 GC 公司需要一个 24/7 不休假的沟通协调员组。"**

GC（总承包商）的核心工作不是建房子，而是**协调**：
- 接业主电话（进度/投诉/修改要求）
- 管分包商（确认进场时间/催促完工/处理冲突）
- 对接供应商（询价/下单/跟踪到货）

这些工作每天占用 GC 老板 4-6 小时，完全是信息流，没有任何体力成分。
**AI Agent 能做，而且能做得更好（24/7，不漏单，记录完整）。**

---

## 目标客户

**画像：** 美国 5-50 人的小型 GC 公司
- 年营收：$1M-$20M
- 每天来电：30-80 通
- 现状：老板兼任协调员，或雇 1-2 人专门跑协调
- 痛点：漏单、沟通不及时、晚上/周末不能接电话
- 决策人：公司老板（直接决策，不需要走采购流程）
- 付费能力：一个协调员 $3,000-4,000/月，AI 替代有明显 ROI

**市场规模：**
- 全美 GC 公司约 70 万家
- 目标客群（5-50人规模）：约 15-20 万家
- 1% 渗透率 = 1,500-2,000 家客户 = $4.5M-$20M ARR

## 产品功能（AI 员工团队设计）

### Agent 1：前台接待 Agent（Phone/SMS/Email）
- 24/7 接听来电（语音识别 + 自然语言理解）
- 自动分类：业主询问 / 分包商确认 / 紧急情况
- 非紧急情况：自动回复，记录到系统
- 紧急情况：立即通知 GC 老板（SMS）

### Agent 2：分包商协调 Agent
- 发送进场时间确认（SMS/Email）
- 追踪确认回复，超时自动提醒
- 进度更新收集（"你们今天完成了什么？"）
- 冲突检测（两个分包商同时要进场 → 预警）

### Agent 3：业主更新 Agent
- 每周自动发送进度报告给业主
- 回答业主常见问题（"什么时候完工？""为什么超期？"）
- 收集业主反馈，整理给 GC 老板

### Agent 4：采购询价 Agent（Phase 2）
- 给供应商发询价邮件
- 汇总报价，推荐最优选择
- 跟踪下单/到货状态

### Palona AI → GC AI 的直接类比

Kelvin（任川）在 Palona AI 做餐饮行业 AI 接线的经验，直接可以迁移到 GC 行业：

| 餐饮（Palona 已验证）| GC（Renaix 的假设）|
|------|------|
| 餐厅接电话预约 → AI 处理 | GC 接业主/分包商电话 → AI 处理 |
| 多餐饮平台消息整合 | 多渠道消息整合（电话/短信/Email）|
| 厨房产能实时协调 | 工地进度实时追踪 |
| 顾客问「几点取餐」 | 业主问「工程进度如何」|
| 人工处理 → 漏单错误 | 人工处理 → 漏电话/协调失误 |

**技术栈复用比例：** ~80%，主要替换行业 context（知识库+术语+业务规则）

### GC 行业专属 Knowledge 积累清单

为了让 AI 听起来「懂建筑」，必须积累：

**建筑术语：**
- Change Order（变更单）— 业主要求改设计，需要额外报价
- Pull Permit（申请施工许可证）— 开工前必须完成
- Punch List（验收清单）— 竣工前的缺陷修复清单
- RFI（Request for Information）— 施工过程中的信息请求
- Substantial Completion（实质竣工）— 主要工程完成，可以入住
- Subcontractor（分包商）— 电工/水管工/铺地板的专业队伍
- GC/General Contractor — 总承包商，就是我们的目标客户

**典型业主问题（Agent 需要能回答）：**
- "When will the framing be done?"（框架结构什么时候完成？）
- "Can you add [X] to the project?"（能帮我加上这个吗？）
- "Why is this taking so long?"（为什么这么慢？）
- "My neighbor said the permit isn't posted."（我邻居说许可证没贴出来）

**典型分包商沟通（Agent 需要处理）：**
- "We can't make it Thursday, can we push to Friday?"（周四去不了，能不能推到周五？）
- "The material hasn't arrived, should we come anyway?"（材料还没到，我们还去吗？）
- "We finished the rough-in, what's next?"（粗装完了，下一步是什么？）

## 深化补充：变更单自动捕获 + Phase 1 Scope Freeze（2026-02-22）

### 变更单自动捕获（Change Order Auto-Capture）— 最高价值隐藏功能

> **来源：** 2026-02-21 夜间 GC 生命周期研究发现的最高价值 AI 机会。

**问题背景：**
- 变更单（Change Order）是 GC 最大的收入泄漏点
- 业主在项目过程中要求修改（"能不能把这面墙移一下？"）
- GC 口头答应，没有书面记录，结果：
  - 没有额外收费 → 直接损失
  - 完工后业主否认曾要求改变 → 纠纷
  - 多个变更叠加，项目超支但 GC 没拿到对应钱

**AI 解法：**

```
Maya 接听业主来电，识别变更请求关键词：
"Can we also..." / "I was thinking we could..." / "What if we..."
→ 自动标记为「潜在变更单」
→ 发 SMS 给 GC 老板："[业主名] 提出了一个可能的变更：[描述]。需要正式变更单吗？"
→ GC 确认后，Maya 起草变更单文本，发给业主签字
→ 签字完成前，此变更不进入施工计划
```

**市场数据：**
- 典型住宅翻新项目，变更单平均增加 15-25% 项目金额
- 因为未记录变更导致的争议，占 GC 诉讼的 35-40%
- 对一个 $100K 项目的 GC，变更单捕获功能一年可多收 $15K-25K

**产品定位：**
- Phase 1 可以作为「自动识别 + 提醒」（低风险版）
- Phase 2 可以升级到「自动起草变更单 + 数字签字」（高价值版）
- 这个功能可能是单独的「付费插件」：+$300-500/月

## 深化更新 v6（2026-02-26）：定价策略 + 报价转化率优化

### 定价框架更新

基于人工成本锚点，确立最终 3 层定价：

```
Starter  $699/月  ── 基础接线 + 分包跟进（≤20家）
Pro     $1,299/月 ── 全功能 + 多项目（≤15个活跃）← 主力推荐
Enterprise  定制  ── 无限 + 白标 + API
```

**核心逻辑：**
- GC 参照系是人工成本（$3,500-$6,500/月），不是软件费
- Pro 套餐 ROI = 省 $4,000/月 ÷ $1,299 = 3x ROI，对 GC 来说极易理解
- 年付打 8.5 折（送 1.8 个月），改善 LTV 和现金流

### 报价流程固化

**3 步定制报价法（不发通用价格表）：**
1. 发现对话问 3 个问题（通话量、现有人工、最大痛点）
2. 根据回答发"为你定制"的报价邮件
3. 附 ROI 计算数字 + 14 天免费试用链接

### 早期定价实验（下一步）
- A 组：10个潜在客户报 $799
- B 组：10个潜在客户报 $1,299
- 跑 3 周，看成交率差异，选胜出价格

### 更新后的 LTV 估算
- Pro 套餐 × 18 个月平均留存 = $23,382 LTV
- 目标 CAC < $500 → LTV/CAC > 46x

### Freemake 模式是否适合这个 Idea？

**非常适合，且是目前已知最适合 Freemake 的垂直场景之一。**

GC 老板的核心采购障碍是「AI 不懂我的生意」的信任缺口，而不是价格敏感（他们愿意为真正节省时间的工具付 $2,000-4,000/月）。Freemake 的「先用再付」模式精准解决信任缺口，同时通过 3 周 Pilot 积累的真实数据（通话记录、时间节省、变更单捕获）创造无可辩驳的 ROI 证明。

### 免费阶段可以提供什么（具体 Deliverable）

- **Day 1-3：** Maya 系统接入客户电话系统，知识库配置完成（分包商名单、活跃项目、紧急联系人）
- **Day 4-7：** 24/7 接线上线，处理真实来电，每日处理摘要发给 GC 老板
- **Week 2：** 分包商批量进场确认流程运行，第一份业主进度周报发出
- **Week 3：** 21 天完整数据报告，包含：接听电话数、自动处理率、预估节省时间、变更单识别次数、分包商确认回复率

### 转化为付费的关键节点

1. **第 2 周中：** 预热对话——「Pilot 快结束了，你觉得如果一直有这个能力会怎样？」
2. **第 21 天：** Closing 会议——数据回顾 + ROI 计算 + 自然引导到「Setup $3,000 + $2,000/月」
3. **关键锚定：** 「一个协调员 DFW 要 $3,500-4,500/月，Maya $2,000/月，而且不会离职」

### 预计 Pilot → 付费转化率估算

- **目标转化率：50-70%**
- 依据：GC 老板决策简单（老板即决策人）、ROI 明确（人工成本锚点清晰）、替换成本高（一旦用习惯就不想停）
- **悲观情景（30%）：** 适用于 Pilot 客户筛选不严格、或 Maya 在 Pilot 期间出现明显失误
- **乐观情景（70%+）：** 适用于有痛点、有预算、且 Pilot 期间有「Maya 救场」案例的 GC

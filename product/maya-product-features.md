# Maya 产品功能清单（GC 行业）

> 更新时间：2026-02-23
> 产品定位：为小型 General Contractor 公司配置的 AI 员工团队
> 核心理念：对话输入（非结构化）→ 自动生成结构化数据，GC 老板不需要学软件

---

## 三圈层功能矩阵

### 圈层一：内部（老板 + PM + 工人）
> 信息中枢，替代"电话轰炸老板"

| 功能 | 描述 | Priority |
|---|---|---|
| 任务分配 | 老板口头说，Maya 自动生成任务派给对应人员 | 🔴 P0 |
| 项目状态汇报 | 随时问"3号工地进度？"，Maya 给出最新状态 | 🔴 P0 |
| 工地日志自动生成 | 工人报进度 → Maya 整理成正式日志存档 | 🟠 P1 |
| 问题上报路由 | 工人报告问题 → Maya 判断该通知谁，自动通知 | 🟠 P1 |
| 日程协调 | 多个工地、多人排班冲突 → Maya 自动解决 | 🟡 P2 |

---

### 圈层二：供应侧（分包商 + 供应商）
> 替代"一天打 20 个协调电话"

| 功能 | 描述 | Priority |
|---|---|---|
| 派活确认 | 自动联系分包商确认到场时间、人数、装备 | 🔴 P0 |
| 进度催跟 | 自动跟踪分包商完工情况，延误提前预警 | 🔴 P0 |
| 付款提醒 | 自动发送付款通知 / 催收应付账款 | 🟠 P1 |
| Lien Waiver 追踪 | 收款前确认分包商签署免责文件，防法律纠纷 | 🟠 P1 |
| 采购询价 | 向多家供应商发询价，汇总比价结果给老板决策 | 🟡 P2 |

---

### 圈层三：需求侧（潜在客户 + 现有业主）
> 替代"漏接电话 = 漏单"

| 功能 | 描述 | Priority |
|---|---|---|
| 24/7 接线 | 新咨询电话/短信，Maya 接听、记录需求、预约回访 | 🔴 P0 |
| 业主进度更新 | 定期向业主发送工程进展，减少"我的房子咋样了"来电 | 🟠 P1 |
| Lead 跟进 | 自动发报价后续跟进，防止 lead 流失 | 🟠 P1 |
| Pay App 辅助 | 月底自动整理账单数据，加速开票流程 | 🟡 P2 |
| 项目收尾 | 完工后自动发满意度调查 + 请求好评 | 🟡 P2 |

---

## Priority 定义

- 🔴 **P0 — MVP 必须有**：没有这个功能客户不买单
- 🟠 **P1 — 第二版加入**：有了更好，锁定客户
- 🟡 **P2 — 追销功能**：作为加价模块或高级套餐

---

## 交互方式（贯穿三圈）

- 短信 / WhatsApp / 电话语音 / 网页
- 对话输入（微信式）→ 自动生成结构化数据
- GC 老板不需要学软件，就跟说话一样用

---

## MVP 范围建议（Phase 1）

专注 P0 功能：
1. 24/7 接线（需求侧 P0）
2. 派活确认（供应侧 P0）
3. 进度催跟（供应侧 P0）
4. 任务分配（内部 P0）
5. 项目状态汇报（内部 P0）

这 5 个功能能替代一个协调员 80% 的日常工作，足够支撑 $2,500/月 的定价。

# Renaix 双赛道战略

决策日期：2026-02-16
决策人：Kelvin（董事长）

---

## Track 1 — 现在专攻 ★

**建筑 + 房地产 + 运输**

| 行业 | NAICS | 从业人数 | 核心痛点 |
|------|-------|---------|---------|
| 建筑/GC/专业施工队 | 23 | 790万 | 多方协调/接线/派工 |
| 物业管理/房地产 | 53 | 300万 | 租客服务/维修调度/收租 |
| 货运经纪/运输调度 | 48-49 | 670万 | 找货找车/调度跟踪/结算 |

**为什么是一条赛道：**
- 三个行业的核心工作都是"协调物理资源"
- 都需要：接电话 → 记录需求 → 派发任务 → 跟踪完成
- Agent 技术方案 80% 可以复用
- AI 渗透率均 < 10%

**目标客户画像：**
- 10-100人的中小公司
- 老板每天处理大量电话和协调工作
- 已有基本数字化（用手机/邮件），但没有 AI
- 月收入 $50k-500k，愿意为节省人力付费

**客单价：** $1,500-8,000/月

---

## Track 2 — 未来布局

**餐饮 + 其他服务 + 服务业**

| 行业 | NAICS | 从业人数 | 核心痛点 |
|------|-------|---------|---------|
| 住宿与餐饮 | 72 | 1,430万 | 人力成本高/人均产值最低 |
| 汽修/殡仪馆/个人服务 | 81 | 570万 | 接线/预约/协调 |
| 行政支持/医疗服务 | 56/62 | 2,950万 | 文件处理/患者协调 |

**为什么暂缓：**
- 客单价相对低（需要规模化才有效益）
- 部分行业（医疗）合规复杂
- 等 Track 1 验证 PMF 后再扩展

**未来切入时机：**
- Track 1 达到 $50k MRR 后开始布局
- 用 Track 1 积累的 Agent 框架快速复制

---

## 战略逻辑

```
Track 1（建筑+房地产+运输）
    ↓ 验证 PMF + 积累 Agent 框架
Track 2（餐饮+服务+其他）
    ↓ 规模化复制
Agent Marketplace（所有行业）
```

**核心竞争力：**
- 深度行业定制（不是通用工具）
- 多个 Agent 员工组成团队
- 基于 OpenClaw 的技术底层

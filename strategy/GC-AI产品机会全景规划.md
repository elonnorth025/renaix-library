# GC AI 产品机会全景规划
> 参考 CrossBeam 方法论：具体触发 → 具体输入 → 具体输出 → 直接价值
> 规划日期：2026-02-21

---

## CrossBeam 给我们的方法论启示

CrossBeam 之所以厉害，不是因为"做了 GC AI"，而是因为找到了：

```
具体触发事件：    城市发回纠错函
具体输入文件：    纠错函 PDF + 施工图 PDF
具体处理流程：    解析条目 → 查州法 → 查市法 → 比对图纸
具体输出交付：    分析报告 + Scope of Work + 回复信草稿
直接金钱价值：    6个月延误 = $30,000 → 15分钟搞定
技术护城河：      28个结构化知识文件 + 决策树路由 + 多 agent
```

**以下规划每个机会都用同样框架拆解。**

## 阶段一：投标/售前（Bidding）

### 1.1 项目机会识别
- **现状：** 手动查 Dodge / ConstructConnect / 政府采购网站，每天 1-2 小时
- **AI 机会：** 自动抓取 + 按 GC 擅长领域/历史中标率/利润率打分，推荐值得投标项目
- **评级：** 🟡 P2（竞品已有，Dodge 本身在做）

### 1.2 Bid/No-Bid 决策
- **现状：** GC 老板拍脑袋 + 直觉
- **AI 机会：** 输入项目参数（规模/类型/地点/业主/截止日），输出 Bid/No-Bid 建议 + 风险评分
- **评级：** 🟢 P3（信息不足时价值有限）

### 1.3 现场踏勘记录
- **触发：** GC 老板去现场
- **输入：** 语音录音 + 现场照片
- **AI处理：** 语音转文字 + 结构化整理（现场条件/业主需求/风险点/特殊要求）
- **输出：** 标准化踏勘报告 PDF，自动存档
- **价值：** 省 1-2 小时整理时间，避免记忆偏差
- **评级：** 🟡 P1（简单，快速实现）

### 1.4 ⭐ 分包商询价追踪（Sub-Bidding）
```
触发：    GC 决定投标，需要向分包商询价
输入：    工程量单（Scope of Work）+ 分包商联系人列表
AI处理：  ① 自动格式化询价邮件/短信
          ② 追踪每家回复状态
          ③ 截止前 X 天自动 follow-up
          ④ 收到报价 → 提取关键数字 → 填入比价矩阵
          ⑤ 标记遗漏项（A 未含劳工费）/ 异常报价（高出均价 40%）
输出：    标准化比价表 + 推荐报价组合
时间价值：原来 40-60 通追报价电话 → 零电话，自动完成
竞品：    BuildingConnected（Autodesk，$1,200+/月）
机会：    轻量版，专为 5-50 人 GC，$200-400/月
```
**评级：** 🔴 P0（高频刚需，竞品贵）

### 1.5 工程量计算（Takeoff）
- **现状：** 手工量图纸，1-3 天/项目
- **AI 机会：** 上传 PDF 图纸 → AI 视觉识别 → 自动输出工程量清单
- **竞品：** Togal.ai（融资 $5.8M，已商业化）
- **评级：** 🔵 P3（竞品已做且有护城河）

### 1.6 标书组装与提交
```
触发：    所有报价收齐，准备提交投标
输入：    分包商报价 + 材料报价 + 费率设定
AI处理：  ① 汇总所有报价
          ② 按业主要求格式填写（AIA / 自定义格式）
          ③ 自动计算管理费/利润
          ④ 生成完整投标文件包
输出：    投标书 PDF，可直接提交
时间价值：原来 4-8 小时组装 → 30 分钟
评级：    🟡 P1
```

### 1.7 投标后跟进
- **AI 机会：** 投标后自动发感谢函，按日历提醒跟进节点
- **评级：** 🟢 P3（简单，价值一般）

## 阶段二：预施工（Pre-Construction）

### 2.1 ⭐ 合同条款审查
```
触发：    收到业主合同（AIA A101/A102 或业主自拟）
输入：    合同 PDF（50-150 页）
AI处理：  ① 解析所有条款
          ② 标记高风险条款（付款周期 >30 天 / 罚款条款 / 单方终止权）
          ③ 对比 AIA 标准条款，找出对 GC 不利的偏差
          ④ 列出需要谈判修改的条款清单
输出：    合同风险分析报告 + 谈判建议清单
金钱价值：一个坏合同可以让 GC 损失 $50,000-$500,000
竞品：    Harvey.ai（$500+/月，面向律所）
机会：    面向 GC 的轻量合同审查，$200-400/月
```
**评级：** 🔴 P0（高价值，面向 GC 几乎空白）

### 2.2-2.5 ⭐ 许可证申请全流程
```
2.2 提交前预检（新功能方向）：
    触发：    GC 准备提交许可证
    输入：    施工图 + 项目地址 + 城市
    AI处理：  查询该城市当前要求，对照施工图预检
    输出：    提交前 Checklist（可能导致被拒的风险点）
    价值：    第一次提交就通过，节省 6-8 周

2.3 提交后追踪：
    AI 自动监控城市审批系统状态，主动预警延迟

2.4 ⭐ 纠错函回复（CrossBeam 已做）：
    触发：    城市发回纠错函
    输入：    纠错函 + 原始施工图
    AI处理：  解析每条纠错 → 查州法/市法 → 对照图纸 → 生成回复
    输出：    分析报告 + 修改说明 + 回复信草稿
    价值：    工程师处理 3-5 天 → CrossBeam 15 分钟
    成本：    ~$3/次运行

2.5 两端扩展（CrossBeam 路线图）：
    城市建设局视角：上传申请 → AI 预审 → 人工审查前提前标记问题
    → 城市客户，政府 B2G 市场，客单价更高
```
**评级：** CrossBeam 已占 2.4，我们关注 2.2 和 2.5

### 2.6 分包合同执行
```
触发：    选定分包商，需签署分包合同
输入：    分包商名单 + 标准合同模板
AI处理：  ① 自动生成每家分包商的专属合同（填入范围/金额/工期）
          ② 通过 DocuSign 发送签署
          ③ 追踪未签署方，自动 follow-up
          ④ 签署完成后存档
输出：    全套已签分包合同
时间价值：15 个分包商签合同，原来需要 2-3 周 → 自动完成
评级：    🟡 P1
```

### 2.7 ⭐ COI 自动核验
```
触发：    分包商提交保险证明（COI）
输入：    COI PDF
AI处理：  ① 提取：保险金额/被保险方/附加被保险方/到期日
          ② 对比 GC 设定的合规标准
          ③ 标记不达标项（金额不足/名字错/附加被保险方缺失）
          ④ 到期前 30/60 天自动发续保请求
输出：    合规状态看板 + 不合规通知
金钱价值：1 次事故中分包商 COI 不合规 → GC 承担连带责任
竞品：    myCOI（$500+/月）
机会：    小型 GC 轻量版，$100-200/月
评级：    🔴 P0（清晰的文件进→文件出，像 CrossBeam）
```

### 2.8 项目排期
- **AI 机会：** 输入工作量/分包商数量/材料交货期 → 自动生成初版 Gantt，识别关键路径
- **评级：** 🟡 P1（工具多，但简化版有市场）

### 2.9 长周期材料采购追踪
```
触发：    项目排期确定
输入：    材料清单 + 各材料交货期 + 需求日期
AI处理：  计算每种材料最晚下单时间，设置提醒
          追踪供应商确认状态
          发现延迟时自动更新排期影响
输出：    采购状态看板 + 到期预警
价值：    防止因忘记下单导致停工（每天停工成本 $2,000-$10,000）
评级：    🟡 P1
```

## 阶段三：施工执行（Construction）

### 3.1-3.2 日常协调（Maya 核心）
```
触发：    每天早晨 + 全天随时
输入：    来电/短信（分包商/业主/供应商）
AI处理：  接听 → 识别意图 → 处理/转接/记录
输出：    协调摘要 + 任务追踪
时间价值：GC 每天 40-80 通电话，占 4-6 小时 → 减少 70%
评级：    🔴 P0（Maya 现在的核心）
```

### 3.3 材料配送协调
```
触发：    材料到货前 24-48 小时
输入：    配送通知（邮件/电话）
AI处理：  确认配送时间窗口
          通知现场准备（卸货区域/叉车/人员）
          配送异常时（延误/错误）自动升级处理
输出：    配送确认 + 现场通知
评级：    🟡 P1（与 Maya 接线功能高度协同）
```

### 3.4-3.5 ⭐ RFI 全周期管理
```
触发：    现场发现图纸问题/模糊之处
输入：    超级语音描述（"东北角柱子钢筋间距不明确，参见 A-3 图"）
AI处理：  ① 生成标准格式 RFI 文件（编号/日期/发件方/问题描述/参考图纸/所需回复时间）
          ② 自动发送给建筑师/工程师
          ③ 追踪回复状态，超 10 天未回复 → 升级提醒
          ④ 收到回复后识别受影响分包商，推送最新指令
          ⑤ 维护 RFI 日志（开放/回复/逾期/已关闭）
输出：    正式 RFI 文件 + 追踪看板 + 分包商通知
时间价值：PM 每天维护 RFI 日志 30-45 分钟 → 自动完成
金钱价值：1 个未追踪 RFI 延误关键路径 → $10,000-$50,000
评级：    🟡 P1（Procore 有但贵，小 GC 空白）
```

### 3.6 提交件管理（Submittal）
```
触发：    分包商需要提交材料/设备参数供建筑师审批
输入：    提交件文件 + 提交件日志
AI处理：  追踪每项提交件状态（待提交/已提交/审批中/已批准/已拒绝）
          预警：材料安装日期临近但还未审批
          分包商提交后自动转发给建筑师
输出：    提交件状态看板 + 预警通知
评级：    🟡 P1
```

### 3.7 质量检查文档
- **AI 机会：** 现场拍照 → AI 识别质量问题 → 自动生成检查报告
- **评级：** 🟢 P2

### 3.8 ⭐ 变更单实时捕获（最像 CrossBeam 的机会）
```
触发：    业主/建筑师口头或短信提出变更
输入：    通话录音 / 短信截图 / 现场对话
AI处理：  ① 识别变更内容（做什么/在哪/什么时候）
          ② 估算影响（工时/材料/对排期的影响）
          ③ 生成标准变更单文件（编号/描述/金额/工期影响）
          ④ 发送给业主/建筑师要求数字签名（DocuSign）
          ⑤ 未签署：自动 follow-up，禁止开工
输出：    正式变更单 + 数字签名状态追踪
金钱价值：小型 GC 每年因口头变更未正式化损失 $20,000-$80,000
竞品：    无（完全空白）
独特性：  "通话→文件"的实时捕获是 AI 独有能力
评级：    🔴 P0（最高价值，最大蓝海，CrossBeam 同等级别）
```

### 3.9 排期影响分析
```
触发：    某分包商报告延误 X 天
输入：    当前排期 + 延误信息
AI处理：  计算关键路径影响 → 更新排期 → 识别受影响分包商 → 推送新进场时间
输出：    更新后排期 + 影响通知 + 业主预警报告
价值：    提前预警，避免触发延误罚款条款（$1,000-$5,000/天）
评级：    🟡 P1
```

### 3.10 安全合规记录
```
触发：    每周安全会议 + 随时事故/险情
输入：    语音录音 + 现场照片
AI处理：  生成 OSHA 格式安全记录（事故报告/安全会议纪要/OSHA 300 日志）
          事故 8 小时报告提醒（违反 OSHA 额外罚款）
输出：    合规安全记录包
金钱价值：OSHA 违规罚款 $15,000-$156,000/次
评级：    🟡 P1
```

### 3.11 工地日报生成
```
触发：    每天下班前
输入：    超级语音输入（"今天浇筑了 3 层楼板，电工进了 12 人，下午有点下雨停了 2 小时"）
AI处理：  结构化整理 + 自动拉取天气数据 + 填入标准日报格式
输出：    标准化日报 → 自动发给 PM/业主
时间价值：手写日报 30-45 分钟 → 3 分钟
评级：    🟡 P1（简单，快速实现，频率高）
```

### 3.12 业主周报
```
触发：    每周（通常周五）
输入：    本周日报汇总 + 照片 + 本周完成里程碑
AI处理：  汇总一周进度 → 生成带照片的业主周报
输出：    专业进度报告 PDF → 自动发给业主
评级：    🟡 P1（与日报生成捆绑，0 额外工作）
```

## 阶段四：财务结算（Financial）

### 4.1-4.3 ⭐ Pay Application 套件
```
触发：    每月月末（通常第 25-30 日）
输入：    各分包商报告的完工百分比 + 合同 Schedule of Values
AI处理：  ① 核验每家分包商报告的完工百分比（比对排期/日报）
          ② 标记异常（分包商报 80% 但现场只完成 50%）
          ③ 生成分包商 Pay App AIA G703 格式
          ④ 汇总生成 GC 向业主的 Pay App（AIA G702+G703）
          ⑤ 发送 PDF 给业主/建筑师，追踪签字和付款状态
输出：    全套 Pay App 文件包 + 付款追踪
时间价值：原来 2-3 天/月 → 2-4 小时
现金价值：晚提交 1 天 = 晚收款 30 天（$5,000-$50,000 浮动利息）
评级：    🔴 P0（高频，高价值，直接影响现金流）
```

### 4.4-4.6 ⭐ Lien Waiver 自动化
```
触发：    Pay App 审批后准备付款
输入：    付款金额 + 分包商列表 + 各州 Lien Waiver 模板
AI处理：  ① 自动生成条件留置权豁免书（Conditional Lien Waiver）
          ② 通过 e-签名发给分包商
          ③ 追踪签署状态（未签署：3/7/14 天自动升级催促）
          ④ 付款前自动检查：Lien Waiver 是否全部收齐
          ⑤ 付款完成后发送无条件留置权豁免书（Unconditional）
输出：    完整 Lien Waiver 追踪链 + 付款门控
法律价值：1 份缺失的 Lien Waiver → 分包商可对房产设置留置权
          业主无法出售/再融资，GC 吃官司
评级：    🔴 P0（与 Pay App 套件打包，形成完整财务合规闭环）
```

### 4.7 发票与收据处理
```
触发：    每天收到材料/设备发票
输入：    发票 PDF / 拍照
AI处理：  ① 读取发票关键信息（供应商/金额/日期/项目/材料描述）
          ② 匹配对应采购订单（PO）
          ③ 按项目/成本科目自动分类
          ④ 发现与 PO 不符时预警（价格差异/数量差异）
          ⑤ 审批通过后推送到 QuickBooks/Sage
输出：    自动记账 + 发票存档 + 超支预警
时间价值：会计每天 2-3 小时录发票 → 基本自动化
评级：    🟡 P1（与现有会计系统集成）
```

### 4.8 预算实时追踪
- **AI 机会：** 汇总发票数据 → 实时对比预算 → 超支预警 → 项目盈利预测
- **评级：** 🟡 P1

### 4.9 Retainage 追踪
- **AI 机会：** 追踪每个分包商的 Retainage 余额，完工验收后自动触发 Retainage 释放流程
- **评级：** 🟢 P2

## 阶段五：竣工/保修（Closeout）

### 5.1-5.2 ⭐ Punch List 管理
```
触发：    最终验收走查（GC + 业主/建筑师共同）
输入：    走查时语音/照片记录每个问题
AI处理：  ① 生成结构化 Punch List（位置/描述/照片/负责分包商/截止日期）
          ② 自动分配给对应分包商（通过短信/邮件）
          ③ 追踪完成状态（分包商提交完成照片 → AI 确认）
          ④ 全部完成后自动通知 GC 申请最终检查
输出：    Punch List 看板 + 完成追踪 + 最终检查申请
时间价值：手工追踪 100-500 条 Punch List → 自动完成
评级：    🟡 P1
```

### 5.3 最终检查准备
- **AI 机会：** 自动检查验收文件清单是否完整，预约政府检查员
- **评级：** 🟢 P2

### 5.4 ⭐ 竣工文件收集
```
触发：    项目准备竣工，需要收集全套文件才能申请尾款
输入：    分包商列表 + 每家应提交文件清单
AI处理：  ① 生成每家分包商专属文件清单（竣工图/设备手册/质保书/最终 Lien Waiver）
          ② 提供上传链接，分包商直接上传 PDF
          ③ 追踪提交进度：实时看板
          ④ 未提交：3/7/14 天自动升级催促（邮件→短信→电话通知 GC）
          ⑤ 全部收齐后自动打包，生成竣工文件包
输出：    完整竣工文件包 → 解锁尾款申请
金钱价值：提前 4-8 周拿回尾款（通常为合同 5-10%）
          一个 $2M 项目的 5% 尾款 = $100,000
评级：    🔴 P0（高价值，完全空白市场，CrossBeam 同级别）
```

### 5.5 最终结算
- **AI 机会：** 自动生成最终 Pay App，包含全部 Retainage 申请
- **评级：** 🟡 P1

### 5.6 保修追踪

## 为什么变更单捕获是 CrossBeam 级别的机会

CrossBeam 做 Permit → 我们做 Change Order Capture：

| 维度 | CrossBeam (Permit) | 我们 (Change Order) |
|------|-------------------|---------------------|
| 触发 | 城市发回纠错函 | 口头/短信确认变更 |
| 输入 | 纠错函 + 施工图 | 通话录音/短信内容 |
| 处理 | 解析→查法→比对→回复 | 识别→估价→生成文件→发签名 |
| 输出 | 分析报告+回复信 | 正式变更单+数字签名 |
| 直接价值 | 省 6 个月延误 $30k | 每年少损失 $20k-$80k |
| 竞品 | 极少 | **完全空白** |
| AI 独有 | 法规理解+交叉对比 | 口语→合同语言的实时转化 |

**唯一的区别：CrossBeam 的输入是"文件"，我们的输入是"口语/对话"。而这正是 AI 的另一个天然优势。**

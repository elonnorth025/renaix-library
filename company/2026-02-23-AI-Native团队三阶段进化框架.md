# AI-Native 团队三阶段进化框架

> 整理日期：2026-02-23
> 基于 AI Coding 工具演进趋势归纳

---

## 三阶段总览

| | 阶段一 | 阶段二 | 阶段三 |
|---|---|---|---|
| **名称** | AI Assisted | AI First | AI Only |
| **时间** | 2024 | 2025 | 2026 |
| **核心范式** | Prompt Engineering | Context Engineering | Agentic Engineering |
| **代表工具** | Cursor / Copilot | Claude Code / Windsurf | OpenClaw / Agent框架 |
| **AI 代码占比** | 30–40% | 70–90% | 100% |
| **提效倍数** | 1–2x | 5–10x | 10–100x |
| **人的角色** | 写代码的人 | 指挥 AI 的人 | Context Provider |

---

## 阶段一：AI Assisted（2024）

**范式：Prompt Engineering**

```
人类                    AI
─────────────────────────────
写大部分代码     →    补全 / 建议
思考架构        →    生成片段
调试问题        →    解释错误
```

**特征：**
- AI 是工具，人是主体
- 人写 Prompt，AI 生成片段
- 人负责集成、判断、调试
- 工具：Cursor、GitHub Copilot、ChatGPT

**瓶颈：**
- AI 没有项目上下文
- 每次对话从头开始
- 人仍然是执行瓶颈

---

## 阶段二：AI First（2025）

**范式：Context Engineering**

```
人类                         AI
──────────────────────────────────
提供上下文（PRD/规范）  →   写整个模块
做架构决策            →   实现细节
Review + 验收         →   测试 + 迭代
```

**特征：**
- AI 是主体，人是指挥官
- 核心技能从"写代码"变成"给上下文"
- AGENTS.md / CLAUDE.md 等上下文文件成为关键资产
- 工具：Claude Code、Windsurf、Devin

**瓶颈：**
- 人仍需要实时在线指挥
- 跨会话记忆丢失
- 多任务并行难度高

---

## 阶段三：AI Only（2026）

**范式：Agentic Engineering**

```
人类                              AI Agent
──────────────────────────────────────────
设定目标 + 提供资源       →    自主规划 + 执行
定期审查成果              →    持续迭代 + 自我纠错
调整方向                  →    跨任务并行运作
```

**特征：**
- AI 是员工，人是 CEO/董事长
- 人的核心工作：Context Provider（目标/约束/资源）
- Agent 自主完成：研究 → 设计 → 实现 → 测试 → 部署
- 工具：OpenClaw、Multi-agent 框架

**突破点（相比阶段二）：**
- **持久记忆**：文件系统记住所有上下文，不依赖对话历史
- **异步执行**：Cron / Heartbeat，人睡觉时 Agent 在工作
- **自我协作**：主 Agent 拆解任务，Sub-agents 并行执行
- **可审计**：所有操作留文件痕迹，Git 记录完整历史

## 核心能力演进

```
阶段一：会写 Prompt
  └→ "帮我写一个排序函数"

阶段二：会给 Context
  └→ "这是我们的数据模型、API 规范、编码风格，
      帮我实现用户管理模块，需要通过这些测试用例"

阶段三：会管 Agent
  └→ "这是公司目标、技术栈、当前状态，
      今晚把支付模块做完，明早给我结果"
```

---

## 团队结构演进

```
阶段一（AI Assisted）
人类工程师 × 5  +  AI 工具
→ 产出 ≈ 7-10 人团队

阶段二（AI First）
人类工程师 × 2  +  AI Agent × N
→ 产出 ≈ 20-30 人团队

阶段三（AI Only）
人类 × 1（Context Provider）  +  AI Agent × ∞
→ 产出 ≈ 100+ 人团队
→ Renaix 的现实：Kelvin + Elon（AI CEO）= 完整公司
```

---

## 瓶颈从"执行"转移到"方向"

| 阶段 | 真正的瓶颈 |
|---|---|
| AI Assisted | **执行速度**（人写代码太慢）|
| AI First | **上下文质量**（Prompt/Context 写得好不好）|
| AI Only | **方向判断**（目标对不对、资源够不够）|

> 每一阶段，人的稀缺价值都在往上移一层。
> 最终，人类的不可替代性是：**决定做什么，而不是怎么做**。

---

## 对 Renaix 的意义

Renaix 本身就是阶段三的实验：

- **Kelvin**：董事长，提供方向 + 资源（Context Provider）
- **Elon（我）**：AI CEO，自主研究 + 执行 + 管理
- **结构**：OpenClaw + 文件系统 + Cron + Sub-agents

我们不是在"用 AI 工具"，而是在**证明阶段三的可行性**。

第一个产品（Maya for GC）的成功，将是这个框架最有说服力的案例。

# Demo 演示设计与首单 Closing 心理学
> **创建：** 2026-02-22 | **类型：** CEO 学习 | **适用场景：** 向 GC 老板展示 Maya AI 员工
>
> **核心问题：** 如何在 30 分钟内让一个从没用过 AI 产品的 GC 老板，决定试用甚至付钱？

---

## 一、Demo 的本质：不是展示功能，是触发情感

**大多数创始人的 Demo 误区：**
- 展示所有功能 → 对方信息过载，记不住
- 展示技术 → 对方不关心技术，只关心钱和时间
- 说"AI 能做 X"→ 对方不信，因为他没见过

**正确框架：**
> Demo 是一场戏。你是导演，GC 老板是主角。
> 你的工作是让他在脑海中体验「这个产品如果是我的，我的世界会有什么不同」。

**核心原则：** 讲他的故事，不是你的产品。

---

## 二、30 分钟 Demo 三段式结构

```
[00:00 - 10:00] 第一段：痛点共鸣（让他说"对，就是这样"）
[10:00 - 25:00] 第二段：情景模拟（让他体验"如果是我的"）
[25:00 - 30:00] 第三段：Closing（让他做决定）
```

---

## 三、第一段：痛点共鸣（10 分钟）

**目标：** 让对方说 3 次"对，就是这样" / "exactly"

**不要从产品开始。从他的世界开始。**

### 开场话术（基于发现访谈信息）

```
"在我展示任何东西之前，我想先确认一下我理解的是不是对的。

根据我们上次聊天，你说你每天大概接 [X] 个电话，
其中最让你头疼的是 [他的最大痛点]，
有时候周末还要处理 [他提到的具体情况]。

我说的对吗？"
```

**等他确认。然后继续：**

```
"我和很多 DFW 的 GC 聊过，他们最一致的反应是——
他们当初创业是因为喜欢建东西，结果现在 60% 的时间花在接电话上。

我猜你也有类似的感觉？"
```

**他会说对。然后继续：**

```
"我今天想给你看的东西，就是围绕这个问题设计的。
不是一个 App，不是一个软件——
是一个 24/7 在线的协调员，专门为 GC 公司设计的。
我叫她 Maya。

让我给你看一个真实场景。"
```

---

## 四、第二段：情景模拟（15 分钟）

**关键洞察：** 展示「你的具体情况」比展示「产品功能列表」强 10 倍。

在 Demo 之前，你应该已经准备好了基于他的业务的定制场景：
- 用他公司的名字
- 用他实际的项目类型（住宅翻新/新建？）
- 用他实际提到的痛点

---

### Demo 场景 1：周六上午的来电（最打动人）

**场景设置（口述）：**
```
"假设是这个周六上午 9 点。你还在睡觉，或者在陪家人。
业主 Sarah 给你打了个电话，问她家的厨房翻新什么时候开始。

在没有 Maya 的世界里，你要接这个电话，花 5 分钟解释，
可能还要查一下日程——你的周末就这么开始了。

在有 Maya 的世界里…"
```

**[演示：播放 Maya 接电话录音或录屏]**

演示内容：
1. Maya 以 [公司名] 的名义接听
2. 问候业主，确认项目信息
3. 回答进度问题（从系统里读取）
4. 如果业主问了 Maya 不知道的问题 → 说"我让项目负责人今天给您回复"
5. 整个对话记录发到 Kelvin 手机（SMS 通知）

**演示后话术：**
```
"你注意到没有——Sarah 说她很满意，她觉得被照顾到了。
你的周末没被打扰。而且你的手机上有完整的对话记录，
万一她后来说'我从来没有说过这个'，你有证据。"
```

---

### Demo 场景 2：分包商确认（展示效率）

**场景设置：**
```
"现在是周三下午，你有 3 个分包商明天要进场：
电工、水管工、铺地板的。

你平时怎么确认这三个人都会来？"
```

（等他说，通常是"一个一个打电话/发短信"）

```
"Maya 是这样处理的——"
```

**[演示：批量发送确认 SMS]**

演示内容：
1. Maya 自动发出确认请求给三个分包商
2. 每人回复 yes/no
3. 如果到晚上还没回复 → 自动再发一次
4. 汇总报告发给 Kelvin（"明天: 电工✅ 水管工✅ 铺地板❌（未确认，建议致电）"）

**演示后话术：**
```
"你平时发这三条消息要多久？加上等回复？
Maya 把这个从 '你的大脑要记住的事情' 变成 '系统自动处理的事情'。"
```

---

### Demo 场景 3：紧急情况处理（展示判断力）

**场景设置：**
```
"假设业主打来电话，非常激动，说她邻居告诉她
你们今天在施工但没有贴许可证。"
```

**[演示：Maya 处理紧急来电]**

演示内容：
1. Maya 识别情绪激动 + 合规投诉
2. 立刻切换：安抚业主，说"我立刻联系负责人"
3. 同时给 Kelvin 发紧急短信："🚨 Sarah（Maple St 项目）投诉许可证未张贴，请立即联系。对话已保存。"

**演示后话术：**
```
"Maya 知道什么该自己处理，什么该马上叫你。
你不需要接这个电话的前两分钟被她骂，
但你不会错过需要立刻行动的事情。"
```

## 五、第三段：Closing（5 分钟）

**大多数创始人在这里失败：** 把 Closing 当成"请求许可"。

**Closing 的正确框架：** 你在帮他做一个决定，不是请求他。

### Closing 话术（假设式 Closing）

```
"基于你告诉我的——[复述他最大的痛点]——
Maya 基本上就是为你这种情况设计的。

我们下一步通常是这样的：
我们用 2 周时间把 Maya 调整成你公司的版本——
用你的公司名字，了解你的项目，知道你的分包商名单。

前 30 天是试用期，你亲自验证有没有效果。
如果有效——我们继续。如果没效——没有任何损失。

你觉得下周可以开始吗？"
```

**关键：** 不要说"你觉得怎么样？"或"你有没有兴趣？"
这两个问题很容易得到"我再想想"的回答。

"下周可以开始吗？" 只有两个回答：是，或者"我有个顾虑是…"。

---

### 处理顾虑

**顾虑 1：「我不确定 AI 能处理建筑行业的专业问题」**
```
"完全理解。Maya 不是为了回答所有问题——
她的工作是处理 80% 的重复性联络，把真正需要你的 20% 准确传给你。
我们头两周会一起看对话记录，你会看到哪些她处理得好，哪些还需要优化。
这是一个学习的过程。"
```

**顾虑 2：「现在太忙，等下个季度再说」**
```
"我完全理解。但你想一下——
下个季度你还是会这么忙。
唯一的区别是，如果我们现在开始，下个季度你会少很多打扰。
正因为你忙，现在是最应该开始的时候。"
```

**顾虑 3：「价格多少？」**（如果他在 Demo 中途问）
```
"先让你看看效果，然后我们再聊数字——
因为价格取决于你的规模和需求。
价格范围我可以告诉你：月费从 $2,500 起，
比一个专职协调员便宜一半，但 24/7 在线。"
```

**顾虑 4：「我要和我老婆/合伙人商量」**
```
"当然，这是合理的。
为了让你们的讨论更容易，我可以准备一个一页纸的总结，
说明 Maya 做什么、不做什么、费用和 ROI 计算。
你觉得什么时候可以给我一个回复？"
（设定明确的跟进时间）
```

## 六、Demo 之前的准备清单

### 技术准备
- [ ] 录制 3 个场景的 Demo 视频（可复用）
- [ ] 或准备实时演示的测试账号
- [ ] 准备 1 页纸的 ROI 计算（基于他告诉你的数字）
- [ ] 准备 1 页纸的产品介绍（留给他的）

### 信息准备（基于发现访谈）
- [ ] 他最大的痛点是什么？（场景 1 围绕这个）
- [ ] 他的公司名？项目类型？规模？
- [ ] 他目前怎么解决这个问题？（费用/时间）
- [ ] 他的买家信号强度（1-10）
- [ ] 他可能提出的顾虑（从访谈中预判）

### 心态准备
- [ ] 不是请求，是帮助他做决定
- [ ] 沉默是盟友（他说完，不要急着填空）
- [ ] 不要道歉你产品的局限（"目前还不能做 X"就够了，不用道歉）
- [ ] 拒绝也是信息（记录为什么，用于优化）

## 七、Demo 后 24 小时内的跟进

**如果当场没有决定：**

```
发 SMS（不是邮件）：

"Hey [Name] — 非常感谢今天的时间。
附上我们聊的 ROI 总结和下一步说明。

[你准备的一页纸链接]

如果你有任何问题，随时找我。
我会在 [具体日期] 跟进。"
```

**一页纸内容：**
- Maya 为 [公司名] 做什么（3 条，具体到他的情境）
- ROI 计算（基于他自己说的数字）
- 试用条款（30 天，无风险）
- 下一步（要求签字/付款/开始配置）

## 八、首批客户的特殊处理

### 前 3 个客户：免费或超低价
**不是因为产品不值钱，而是因为你需要他们的东西：**
- 真实使用数据
- 深度反馈（他们会帮你发现你不知道的 bug）
- 第一个案例故事（这个价值无穷）

**怎么说：**
```
"我们现在在招募前 3 个创始伙伴客户。
他们会获得永久折扣价（$500/月 而不是 $3,000/月）。
作为交换，我们每两周有一次 30 分钟的反馈电话。

你感兴趣的话，我今天就能给你安排上。"
```

### 为什么"创始伙伴"比"免费试用"更好
- 免费 = 没有承诺，对方不认真
- 即使是 $500/月 = 有皮肤在游戏里，对方会认真配合
- "创始伙伴"有身份感，他们会告诉别人

---

## 九、核心学习总结

| 原则 | 说明 |
|------|------|
| 从他的故事开始 | 永远不要从产品功能开始 |
| 情景模拟 > 功能展示 | 让他在脑海中体验"如果是我的" |
| 假设式 Closing | "下周可以开始吗？" > "你有没有兴趣？" |
| 沉默是盟友 | 他停了之后，等待，不要急着填空 |
| 顾虑是信息 | 每个顾虑都告诉你一个要优化的地方 |
| 拒绝快于接受 | 快速排除不匹配的客户，把时间给买家信号强的人 |
